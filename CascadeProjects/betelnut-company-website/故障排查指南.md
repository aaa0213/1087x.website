## 🛠️ 常见问题和解决方案

### 问题1：文件没有正确移动
**解决方案**：
```bash
# 重新移动文件
rm -rf /var/www/1087x/*
cp -r /tmp/betelnut-company-website/* /var/www/1087x/
chown -R www-data:www-data /var/www/1087x
chmod -R 755 /var/www/1087x
```

### 问题2：Nginx配置问题
**解决方案**：
```bash
# 重新创建Nginx配置
rm -f /etc/nginx/sites-enabled/1087x
cat > /etc/nginx/sites-available/1087x << 'EOF'
server {
    listen 80;
    server_name 1087x.139.199.1.35;
    root /var/www/1087x;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
EOF

ln -sf /etc/nginx/sites-available/1087x /etc/nginx/sites-enabled/
nginx -t && systemctl restart nginx
```

### 问题3：端口冲突
**解决方案**：
```bash
# 检查端口占用
netstat -tlnp | grep :8001

# 如果有冲突，使用其他端口
cd /var/www/1087x
nohup python3 -m http.server 8002 &
```

### 问题4：防火墙阻止
**解决方案**：
```bash
# 开放端口
ufw allow 8001
ufw allow 80
iptables -A INPUT -p tcp --dport 8001 -j ACCEPT
```
